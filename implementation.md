## 🗄️ Database Schemas & Queries (`db/database.ts`)

- **`records`**: Stores finalized, user-reviewed structured records (expenses, tasks, reminders).
- **`raw_captures`**: Stores transcripts generated by Whisper before SLM processing.
- **`benchmarks`**: Stores timed parameters, CPU/RAM estimates, success metrics, and `measurement_type` (marking estimated vs measured metrics). Supports safe migration of the `measurement_type` column at startup.
- **`user_profile`**: Stores settings, name, and downloaded model metadata.
- **`getBenchmarkSummary()`**: Computes average latencies and validation success rates dynamically using pure SQL.

---

## 🎙️ Local AI Engines

### 1. Whisper speech transcriber (`services/transcriptionEngine.ts`)
- Uses `whisper.rn` binding to run a local `ggml-tiny.en.bin` model (~75MB).
- Runs 100% on CPU locally to convert audio WAV files to text.
- Times the duration of the transcription and logs benchmark entries.

### 2. Llama schema extractor (`services/extractionEngine.ts`)
- Uses `llama.rn` binding to run a local `qwen2.5-0.5b-instruct-q4_k_m.gguf` model (~390MB).
- Prompts the model to format unstructured text into precise JSON format.
- Automatically handles up to **2 corrective retries** with error descriptions on validation failures.

---

## 🔒 Security Gate & Auditing (`services/networkGuard.ts`)
- Implements strict network filters. All remote calls are blocked by default.
- Gating is temporarily lifted (`isModelDownloadPhase = true`) only during initial onboarding when downloading Whisper and Qwen models from Hugging Face.
- Restricts all HTTP fetch calls in the codebase to allowed Hugging Face subdomains.
- Includes a settings page self-test to verify that traffic blockages are active.

---

## 🛡️ Validation & Graceful Fallbacks (`services/schemaValidator.ts`)
- Implements `validateRecord` to run structure and type-specific rules.
- Handles `retryWithCorrection` to feed extraction error descriptions back to the SLM.
- Generates default low-confidence tasks using `buildFallbackRecord` on terminal failure.

## 📊 Completed UI Screens & Navigation

1. **Onboarding Flow**:
   - `welcome.tsx`: Personalizes the experience.
   - `model-download.tsx`: Connects to NetInfo to verify WiFi, downloads models sequentially with progress bars and step labels, locks the network gate permanently on completion, and cannot be skipped.
2. **Home Screen**:
   - Two large options: "🎙️ Speak a note" (record.tsx) and "⌨️ Type a note" (review.tsx).
   - Recent 5 records preview list.
   - Bottom tab bar navigation (Home, List, Metrics, Settings).
3. **Record Screen**:
   - Waveform/pulse animation while recording audio.
   - On stop: shows "Transcribing locally..." spinner.
   - Decoupled pipeline: only transcribes text using Whisper and redirects to the review page.
4. **Review Screen**:
   - Shows raw transcript in an editable text field.
   - "Extract Structure" button calls Qwen SLM locally with a "Processing locally..." spinner.
   - Displays structured fields as an editable form.
   - Displays a yellow banner (`⚠️ Low confidence`) if confidence is low.
   - Save button updates or inserts records and navigates to the list.
5. **List Screen**:
   - Grouped list of all records with icons (💰 expense, ✅ task, ⏰ reminder).
   - Swipe-to-delete gesture powered by `react-native-gesture-handler`.
   - Tap to edit existing records.
6. **Benchmarks Screen**:
   - Summary cards of latencies, success rates, and total runs.
   - Dynamic horizontal bar chart illustrating latency per stage for the last 8 runs.
   - "Export Benchmark Report" button saves a JSON file and shares it via the native share sheet.
7. **Settings Screen**:
   - Wipe data options, model size indicators, name edits, and air-gap network self-tests.
